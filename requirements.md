Esse sistema sera o backend do Styttech Scheduler, responsável por gerenciar os agendamentos e disponibilizar uma API para o frontend consumir. Ele será desenvolvido utilizando Spring Boot, seguindo a arquitetura hexagonal para garantir 
uma estrutura modular e de fácil manutenção. 

A estrutura do projeto deve seguir o exemplo do módulo de CRM do SytTechPortalService, com pastas para domain, 
ports, use cases, adapters e config. 
O módulo deve ser declarado no `package-info.java` com as dependências necessárias.
